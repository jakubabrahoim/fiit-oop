# Assignment 1 - Specification

<Callout type='warn'>**Submission deadline: 08.03.2026 23:59**</Callout>

## Allowed Frameworks

For game development, you can use these frameworks:

- [LibGDX](https://libgdx.com/) <ExternalLink className="inline w-4 h-4 ml-1 align-text-top" /> - Cross-platform game development framework
- [dyn4j](https://dyn4j.org/) <ExternalLink className="inline w-4 h-4 ml-1 align-text-top" /> - Physics engine for Java

## Assignment Tasks

### 1. Project Intent

Describe:

- What type of game you will create
- What the game is about
- When the player wins
- Game mechanics
- Characters
- Levels
- Objects in the game

### 2. Functional Requirements

- List basic areas/sections that the project will have
- Write tasks that need to be done for each
- Tasks ordered by priority
- You can use Jira or similar tool

<Callout type='info'>
    These won't be complete functional requirements, rather a backlog of tasks
    that you should list.
</Callout>

### 3. UML Class Diagram

Create a diagram of the most important classes - those that best express the core idea of the game.

### 4. UI Sketch

Simple shape-up design according to example: [Shape Up - Chapter 2](https://basecamp.com/shapeup/1.1-chapter-02) <ExternalLink className="inline w-4 h-4 ml-1 align-text-top" />

<Callout type='info'>
    No wireframes or Figma designs. It should be a simple sketch.
</Callout>

## Documentation Must Include

1. Description of the general topic
2. What game you will create
3. Simple UI image
4. List of Entities
