# Seminar 4 - Design Patterns

## Lecture Topics

### Design Patterns

#### Singleton

- Private constructor
- Single instance in the entire application

```java
public class GameManager {
    private static GameManager instance;

    private GameManager() {}

    public static synchronized GameManager getInstance() {
        if (instance == null) {
            instance = new GameManager();
        }
        return instance;
    }
}
```

#### Observer

- Notification about changes
- Publisher-Subscriber pattern

#### Strategy

- Algorithm selection at runtime
- Encapsulation of algorithms

#### Decorator

- Extending functionality without inheritance
- Wrapping objects

### When to use `static` and `synchronized`

- `static` - shared between all instances
- `synchronized` - thread-safe access
