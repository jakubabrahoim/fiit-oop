# Seminar 3 - Feedback & Interfaces

## Lecture Topics

### Interfaces

- **Interface** - defines a contract that a class must implement
- **Abstract class** - can contain implementation and abstract methods

### Comparison: Interface vs. Abstract Class

| Interface                                    | Abstract Class               |
| -------------------------------------------- | ---------------------------- |
| Only method signatures (default from Java 8) | Can have implementation      |
| Class can implement multiple                 | Class can extend only one    |
| No constructor                               | Has constructor              |
| All methods are public                       | Can have different modifiers |

### Object.toString()

- Every class extends `Object`
- Override `toString()` for better output

```java
@Override
public String toString() {
    return "Player{name='" + name + "', score=" + score + "}";
}
```

### IDE Navigation

- **Ctrl+click** - go to definition
- **Ctrl+B** - go to definition
- **Ctrl+Alt+B** - go to implementation
